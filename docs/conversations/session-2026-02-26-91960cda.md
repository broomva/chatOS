> | Observation types | `packages/types/src/state.ts` | `metric | event | error | decision` taxonomy |
> 
> **What's captured today**: model ID, token usage (input/output), finish reason, tool call count, platform, response length — all persisted as JSON observations.
> 
> ---
> 
> **Gaps Identified**
> 
> | Gap | Impact | Priority |
> |---|---|---|
> | **`experimental_telemetry` not enabled** on `streamText`/`generateText` calls | No OTel spans emitted — AI SDK tracing is opt-in and currently off | **Critical** |
> | **No `instrumentation.ts`** / `@vercel/otel` setup | No OpenTelemetry SDK bootstrap in Next.js or bot | **Critical** |
> | **No distributed trace propagation** | Cannot correlate spans across web → bot → external services | High |
> | **Observation types not OTel-aligned** | FS observations use custom schema, not `gen_ai.*` semantic conventions | High |
> | **No LLM cost model** | Token counts exist but no cost attribution | Medium |
> | **No TTFT / throughput metrics** | `msToFirstChunk`, `avgCompletionTokensPerSecond` not tracked | Medium |
