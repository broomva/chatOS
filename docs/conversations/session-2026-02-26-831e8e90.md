> **chatOS Observability & LLM Ops Plan**
> 
> **Context**
> 
> chatOS has a well-designed filesystem-based observation layer (`@chatos/state`) that records agent lifecycle events, messages, token usage, and tool calls to `.agent/` directories. However, it lacks OpenTelemetry instrumentation, distributed tracing, and integration with LLM ops platforms. The AI SDK v6 provides built-in `experimental_telemetry` support that emits OTel spans — but it's currently disabled on all 4 `streamText` call sites. This plan adds best-in-class observability with Langfuse, LangSmith (with thread grouping), and MLflow support while keeping everything opt-in and backwards compatible.
