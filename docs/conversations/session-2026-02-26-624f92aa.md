> The problem is clear. The TUI's `dev` script uses `bun --watch`, which conflicts with pi-tui's raw terminal mode. `--watch` takes control of stdin/stdout for file-change detection and process restart, which prevents the TUI from properly reading keyboard input and writing escape sequences.
> 
> There are two issues:
> 
> 1. **`bun --watch` + raw terminal TUI = broken** — `--watch` intercepts stdin and restarts the process on file changes, corrupting terminal state
> 2. **`turbo dev` runs all apps** — even if run from root, turbo will launch the TUI inside its subprocess manager, where raw terminal I/O doesn't work
> 
> The fix: change `dev` to a watch-build (compatible with turbo), and add a dedicated `tui` script for interactive use.
