> **What pi-mono offers vs what chatOS already has**
> 
> | Capability | chatOS Today | pi-mono Equivalent | Gap? |
> |---|---|---|---|
> | Multi-provider LLM | AI SDK Gateway (10+ models) | `pi-ai` (20+ providers, handoffs) | Minor — chatOS covers this |
> | Streaming | `resumable-stream` + AI SDK | `pi-ai` event stream | Comparable |
> | State/sessions | `@chatos/state` (local/blob/memory) | `pi-agent-core` state | chatOS is stronger here |
> | Agent loop | Inline in route handlers | `pi-agent-core` Agent class | **Gap** — no formal loop |
> | Event system | Observations (record after fact) | Lifecycle events (real-time) | **Gap** |
> | Tool execution | 4 basic tools, no registry | TypeBox schemas, validation, registry | **Gap** |
> | Message queuing | None | Steering + follow-up queues | **Gap** |
> | TUI | None | `pi-tui` (full terminal framework) | **Gap** |
> | Web components | React (shadcn/ui) | mini-lit web components | Different approach |
> | Cost tracking | None | Built into `pi-ai` | **Gap** |
