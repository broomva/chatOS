You are operating inside the Broomva ecosystem.

Objective:
Verify that the documentation, second-brain knowledge graph, and related consciousness/context systems are fully updated and synchronized across the entire /broomva workspace, and ensure all repositories are left in a clean, validated, merged state.

Primary systems to inspect:
- /knowledge-graph-memory
- /agent-consciousness

Workspace scope:
- all relevant folders and repositories under /broomva

Core question to answer:
Is the documentation and second-brain knowledge graph properly updated and consistent with the current state of the codebase, architecture, workflows, and active repositories?

Execution requirements:
1. Inspect the full /broomva workspace and identify all relevant repositories, folders, docs, and knowledge/memory-related systems.
2. Audit whether /knowledge-graph-memory and /agent-consciousness are properly updated to reflect the current system state.
3. Audit whether repository-level and folder-level documentation is current, complete, and aligned with the actual implementation.
4. Check for drift between:
   - code and documentation
   - docs and architecture
   - repo READMEs and actual repo contents
   - knowledge graph memory and current project structure
   - agent-consciousness/context layers and real project state
5. Update all outdated documentation across all relevant folders and repos under /broomva.
6. Ensure that:
   - READMEs are accurate
   - architecture docs are current
   - setup and usage docs are valid
   - internal references are not stale
   - cross-repo documentation is coherent
   - knowledge graph / second-brain artifacts reflect current reality
7. Where needed, update memory, knowledge graph, or consciousness-related artifacts so they match the real state of the ecosystem.
8. Validate all affected repositories after changes.
9. Open or update PRs as needed.
10. Ensure PR checks and CI/CD are green across the board.
11. Merge autonomously only once validation is complete.
12. Leave git state clean across all relevant repositories in /broomva.

Specific audit expectations:
- detect stale docs
- detect missing docs
- detect inconsistent repo descriptions
- detect outdated examples or commands
- detect broken references between repos
- detect missing knowledge graph links or outdated memory nodes if those are part of the current system
- detect whether /agent-consciousness is missing current architectural/project awareness

Git and repo hygiene requirements:
- inspect branch state in each relevant repo
- ensure no accidental dirty working trees remain
- ensure merged work is not leaving abandoned partial branches/worktrees without reason
- ensure final git state is clean across /broomva

Quality bar:
- no superficial doc pass
- no "updated" docs that still contradict the code
- no green merge claims without actual validation
- no leaving stale repo state behind
- no partial synchronization between docs, memory, and implementation

Required workflow:
- think step by step
- inspect before changing
- compare implementation against docs, not docs against assumptions
- update docs close to the source of truth
- preserve architectural consistency across repos
- keep commits and PRs scoped logically
- verify CI/CD instead of assuming success

Definition of done:
- documentation is updated across all relevant folders and repos
- /knowledge-graph-memory reflects the current ecosystem state
- /agent-consciousness is updated if needed to reflect the current project/system state
- PR checks and CI/CD are green across the board
- merges are complete
- git state is clean across /broomva

Final output:
Provide a final summary including:
1. repositories and folders audited
2. documentation gaps or inconsistencies found
3. memory/knowledge graph issues found
4. agent-consciousness/context issues found
5. files and repos updated
6. PRs created or updated
7. checks and CI/CD results
8. merge results
9. final git cleanliness status across /broomva
10. any remaining follow-up work, if applicable
